FlowSynx in 5 Minutes (Quickstart)

This quickstart guide enables you to install FlowSynx, create a minimal workflow, and run it end-to-end in under five minutes. It is designed for first-time users with no prior knowledge of FlowSynx.

1. Install FlowSynx‚Äã

  • Docker (One-Liner)
  • Pre-Built Binary

The fastest way to run FlowSynx is via Docker.
The following command starts FlowSynx with all required services:

docker run -d --name flowsynx \
  -p 6262:6262 \
  -v flowsynx_data:/app/data \
  flowsynx/flowsynx:1.2.4-linux-amd64 --start

What this command does:

  • Pulls the latest FlowSynx image
  • Exposes API on port 6262
  • Persists workflow and execution data in a local Docker volume

Prefer to run FlowSynx locally?
Download a pre-built binary for your OS from the latest release:

Releases

After extracting the ZIP file, start FlowSynx with:

flowsynx --start

After the container starts, FlowSynx is available at:

http://localhost:6262

2. Create a Minimal ‚ÄúHello World‚Äù Workflow‚Äã

Workflows in FlowSynx are defined in JSON.
The following example writes a text file results/test.txt containing the message "Hello, FlowSynx!".

Create a file named:

hello-world.json

Paste the following content:

{
  "name": "Hello World Workflow",
  "description": "A minimal FlowSynx workflow example",
  "tasks": [
    {
      "name": "Hello world",
      "type": { 
        "plugin": "FlowSynx.Storage.Local",
        "version": "1.0.0"
      },
      "parameters": {
        "operation": "write",
        "path": "/app/data/test.txt",
        "data": "Hello, FlowSynx!",
        "overwrite": false
      },
      "dependencies": [],
    }
  ]
}

3. Execute the Workflow‚Äã

You can execute the workflow using either the FlowCtl CLI or the REST API.

Option A ‚Äî Run with flowctl (recommended)‚Äã

If you have installed the CLI:

flowctl workflows executions execute -f ./hello-world.json

Execution begins immediately.

Flowctl

If flowctl is not installed, please refer to Install flowctl guide to help you install it.

Option B ‚Äî Run via REST API‚Äã

1. Add the workflow:‚Äã
curl -X POST http://localhost:6262/workflows \
  -H "Content-Type: application/json" \
  --data @hello-world.json

Copy the returned id value.

2. Execute the workflow:‚Äã
curl -X POST http://localhost:6262/workflows/<WORKFLOW_ID>/executions

4. Verify the Output‚Äã

After successful execution, a file is created at:

/app/data/test.txt

The file contains:

Hello, FlowSynx!

This confirms that your FlowSynx environment and workflow engine are working correctly.
