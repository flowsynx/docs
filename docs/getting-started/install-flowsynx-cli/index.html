Install the FlowSynx CLI

Install the FlowSynx CLI as the main tool for running FlowSynx-related tasks

Step 1: Install the FlowSynx CLI​

  • Linux
  • Windows
  • MacOS

Install from Terminal​

Install the latest Linux FlowSynx CLI to /usr/local/bin:

wget -q https://raw.githubusercontent.com/flowsynx/cli/master/install/install.sh -O - | /bin/bash

Installing a specific CLI version​

The following example shows how to install FlowSynx CLI version 0.1.0.

wget -q https://raw.githubusercontent.com/flowsynx/cli/master/install/install.sh -O - | /bin/bash -s 0.1.0

Install without sudo​

If you do not have access to the sudo command or your username is not in the sudoers file, you can install FlowSynx to an alternate directory via the FLOWSYNX_INSTALL_DIR environment variable. This directory must already exist and be accessible by the current user.

wget -q https://raw.githubusercontent.com/flowsynx/cli/master/install/install.sh -O - | FLOWSYNX_INSTALL_DIR="$HOME/flowsynx" /bin/bash

Installing a specific FlowSynx version without sudo​

The following example shows how to install FlowSynx CLI version 0.1.0.

wget -q https://raw.githubusercontent.com/flowsynx/cli/master/install/install.sh -O - | FLOWSYNX_INSTALL_DIR="$HOME/flowsynx" /bin/bash -s 0.1.0

Install from Powershell​

Install the latest windows FlowSynx cli to $Env:SystemDrive\flowsynx and add this directory to the User PATH environment variable:

iwr -useb https://raw.githubusercontent.com/flowsynx/cli/master/install/install.ps1 | iex

Installing a specific CLI version​

The following example shows how to install CLI version 0.1.0.

$script=iwr -useb https://raw.githubusercontent.com/flowsynx/cli/master/install/install.ps1; $block=[ScriptBlock]::Create($script); invoke-command -ScriptBlock $block -ArgumentList "0.1.0"

Install without administrative rights​

If you do not have admin rights, you can install FlowSynx to an alternate directory via the FLOWSYNX_INSTALL_DIR environment variable. The script below creates the directory if it does not exist.

$Env:FLOWSYNX_INSTALL_DIR = "<your_alt_install_dir_path>"
$script=iwr -useb https://raw.githubusercontent.com/flowsynx/cli/master/install/install.ps1; $block=[ScriptBlock]::Create($script); invoke-command -ScriptBlock $block -ArgumentList "", "$Env:FLOWSYNX_INSTALL_DIR"

Installing a specific CLI version without administrative rights​

The following example shows how to install CLI version 0.1.0.

$Env:FLOWSYNX_INSTALL_DIR = "<your_alt_install_dir_path>"
$script=iwr -useb https://raw.githubusercontent.com/flowsynx/cli/master/install/install.ps1; $block=[ScriptBlock]::Create($script); invoke-command -ScriptBlock $block -ArgumentList "0.1.0", "$Env:FLOWSYNX_INSTALL_DIR"

Install from Terminal​

Install the latest Darwin FlowSynx CLI to /usr/local/bin:

curl -fsSL https://raw.githubusercontent.com/flowsynx/cli/master/install/install.sh | /bin/bash

Installing a specific CLI version​

The following example shows how to install CLI version 0.1.0.

curl -fsSL https://raw.githubusercontent.com/flowsynx/cli/master/install/install.sh | /bin/bash -s 1.12.0

Install without sudo​

If you do not have access to the sudo command or your username is not in the sudoers file, you can install FlowSynx to an alternate directory via the FLOWSYNX_INSTALL_DIR environment variable. This directory must already exist and be accessible by the current user.

curl -fsSL https://raw.githubusercontent.com/flowsynx/cli/master/install/install.sh | FLOWSYNX_INSTALL_DIR="$HOME/flowsynx" /bin/bash

Installing a specific CLI version without sudo​

The following example shows how to install CLI version 0.1.0.

curl -fsSL https://raw.githubusercontent.com/flowsynx/cli/master/install/install.sh | FLOWSYNX_INSTALL_DIR="$HOME/flowsynx" /bin/bash -s 1.12.0
Register path to the OS Environment Varables

In linux-based operating system, if you use the Install without sudo option, you must register the $HOME/flowsynx to the PATH of Environment Variables. To do this, you can use below command:

export PATH=$PATH:$HOME/flowsynx/flowsynx

Step 2: Verify the installation​

Verify the CLI is installed by restarting your terminal/command prompt and running the following:

synx -h
